    Mariadb安装方式

    1、源代码：编译安装
    2、二进制格式的程序包：展开至特定路径，并经过简单配置后即可使用
    3、程序包管理器管理的程序包
    CentOS 安装光盘
    项目官方：https://downloads.mariadb.org/mariadb/repositories/
    国内镜像：
    https://mirrors.tuna.tsinghua.edu.cn/mariadb/yum/
    https://mirrors.tuna.tsinghua.edu.cn/mysql/yum/

    rpm包安装mysql

    rpm -ivh mariadb-server 服务端包
rpm -ivh mariadb 客户端包

yum install mariadb-server mariadb

    二进制格式安装

    二进制包安下载地址：https://mariadb.org/download/
    二进制包，简单来说相当于windows绿色软件，其实就是源码包编译后文件
    官方安装文档，可参考包中的README-wsrep文件
    1、创建用户

    useradd -s /sbin/nologin -r mysql

    2、准备二进制程序

    tar xf mariadb-10.5.5-linux-systemd-x86_64.tar.gz -C /usr/local
cd /usr/local/
ln -s mariadb-10.5.5-linux-systemd-x86_64 mysql
chown root.root mysql -R

    3、修改PATH变量

    cat /etc/profile.d/env.sh
PATH=/usr/local/mysql/bin:$PATH

source  /etc/profile

    4、准备数据目录

    mkdir /data/mysql -pv
chown mysql.mysql /data/mysql

    源码编译安装

    1、安装依赖包

    yum install bison bison-devel zlib-devel libcurl-devel libarchive-devel boost-devel gcc gcc-c++ cmake ncurses-devel gnutls-devel libxml2-devel openssl-devel libevent-devel libaio-devel

    2、创建用户及数据目录

    useradd -s /sbin/nologin -r mysql
mkdir /data/mysql
chown mysql.mysql /data/mysql
tar xf mariadb-10.5.5.tar.gz
cd mariadb

    3、cmake编译安装
    cmake的重要特性之一是其独立于源码(out-of-source)的编译功能，即编译工作可以在另一个指定的目录中而非源码目录中进行，这可以保证源码目录不受任何一次编译的影响，因此在同一个源码树上可以进行多次不同的编译，如针对于不同平台编译
    编译参数：https://dev.mysql.com/doc/refman/5.7/en/source-configuration-options.html
    编译注意下看下结果有没有报错，如果有报错，解决下依赖，执行rm -f CMakeCache.txt，重新编译

    cmake . \
-DCMAKE_INSTALL_PREFIX=/usr/local/mysql \
-DMYSQL_DATADIR=/data/mysql/ \
-DSYSCONFDIR=/etc/mysql \
-DMYSQL_USER=mysql \
-DWITH_INNOBASE_STORAGE_ENGINE=1 \
-DWITH_ARCHIVE_STORAGE_ENGINE=1 \
-DWITH_BLACKHOLE_STORAGE_ENGINE=1 \
-DWITH_PARTITION_STORAGE_ENGINE=1 \
-DWITHOUT_MROONGA_STORAGE_ENGINE=1 \
-DWITH_DEBUG=0 \
-DWITH_READLINE=1 \
-DWITH_SSL=system \
-DWITH_ZLIB=system \
-DWITH_LIBWRAP=0 \
-DENABLED_LOCAL_INFILE=1 \
-DMYSQL_UNIX_ADDR=/data/mysql/mysql.sock \
-DDEFAULT_CHARSET=utf8 \
-DDEFAULT_COLLATION=utf8_general_ci

    4、准备环境变量

    echo 'PATH=/usr/local/mysql/bin:$PATH' > /etc/profile.d/mysql.sh
. /etc/profile.d/mysql.sh

    • mysql用户账号由两部分组成：'USERNAME'@'HOST'

    mysql 客户端

    交互式模式：可运行命令有两类：
    客户端命令：

    \h, help
\u，use
\s，status
\!，system

    服务器端命令：
    SQL语句， 需要语句结束符；
    脚本模式：

    mysql –uUSERNAME -pPASSWORD < /path/somefile.sql
mysql> source /path/from/somefile.sql

    mysql客户端可用选项

    -A, --no-auto-rehash 禁止补全，**建议开启，关闭则会提升性能，但操作不便**
-u, --user= 用户名,默认为root
-h, --host= 服务器主机,默认为localhost
-p, --passowrd= 用户密码,建议使用-p,默认为空密码
-P, --port= 服务器端口
-S, --socket= 指定连接socket文件路径
-D, --database= 指定默认数据库
-C, --compress 启用压缩
-e “SQL“ 执行SQL命令
-V, --version 显示版本
-v --verbose 显示详细信息
--print-defaults 获取程序默认使用的配置

    socket地址

    服务器监听的两种socket地址：
    ip socket: 监听在tcp的3306端口，支持远程通信
    unix sock: 监听在sock文件上，仅支持本机通信，不会走TCP
    如：/var/lib/mysql/mysql.sock
    说明：host为localhost,127.0.0.1时自动使用unix sock

    小结

    常用命令

    查看数据库
MariaDB [(none)]> show databases;
+--------------------+
| Database           |
+--------------------+
| information_schema | #内存数据为库，假的不真实
| mysql              | #用户相关的
| performance_schema | #性能相关的
+--------------------+

一般备份 preformance_schema 和mysql数据库，以及所需要的数据库
脚本技巧， -+不会写入到文件中
mysql -e "show databases" |grep -Ev "test|Database|information_schema"

数据库服务是否运行 
mysqladmin ping 
echo $?

关闭数据库服务
mysqladmin -uroot -pxxx shutdown

查看数据库的状态
mysqladmin -uroot -pxx status

查看当前数据库的端口号，变量
[root@localhost ~]# mysql -e "show variables like 'port';"
+---------------+-------+
| Variable_name | Value |
+---------------+-------+
| port          | 3306  |
+---------------+-------+

在数据库可以调用系统命令
MariaDB [(none)]> system ifconfig

select user();

注意：匿名方式
匿名方式若允许的情况下,是可以登录的，这种方式很危险
mysql -uxxxxxxx 
mysql -uuser11111 

select user,host,password from user;
user字段是空的，表示是匿名用户，使用一般建议删除，为了安全

    客户端配置文件

    mysql 交互式命令
\R 修改提示符  可以man mysql查看帮助文档 

修改提示符
1、修改配置文件
vim /etc/my.cnf.d/mysql-clients.cnf 
[mysql]
prompt="\\r:\\m:\\s \N [\d]> "
2、修改环境变量
export MYSQL_PS1="\\r:\\m:\\s \N [\d]> "
3、进入数据库中修改数据库中的变量
mysql --prompt "\\r:\\m:\\s \N [\d]> "

查看配置文件中的默认选项 /etc/my.cnf.d/mysql-clients.cnf 
mysql --print-defaults 

注意
mysql登录密码写在这里很不安全，测试时可以使用
vim /etc/my.cnf.d/client.cnf
user=root
password=password123

    适用于学习，测试阶段，生产不要使用，相当于删库跑路
    删除目录

    rm -rf /var/lib/mysql/*
systemctl restart mariadb
