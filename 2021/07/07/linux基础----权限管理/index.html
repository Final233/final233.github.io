    Linux用户和组的主要配置文件

    /etc/passwd：用户及其属性信息(名称、UID、主组ID等）

    man 5 passwd
    用户名:X:UID:GID:描述信息（默认为空）:家目录:使用的shell路径
    passwd文件格式
    login name：登录用名（root）username
    passwd：密码 (x) 早期系统版本用户密码存放在/etc/passwd文件中 ,后期存放到/etc/shadow文件中
    UID：用户身份编号 (0) 普通用户默认从1000开始，递增，
    GID：登录默认所在组编号 (0)
    GECOS：默认是用户全名 可设置描述信息
    home directory：用户主目录 (/root) 普通用户家目录/home/username
    shell：用户默认使用shell (/bin/bash) 可指定允许登陆系统 /sbin/nologin

    /etc/group：组及其属性信息

    群组名称：就是群组名称
    群组密码：通常不需要设定，密码是被记录在 /etc/gshadow
    GID：就是群组的 ID
    以当前组为附加组的用户列表(分隔符为逗号)

    /etc/shadow：用户密码及其相关属性

    登录用名
    用户密码:一般用sha512加密 早期使用的MD5 128位加密 $6$xxxx$xxxxxxxxxxxxx $6 sha512 $xxxxxx 随机数 $xxxxxxx 密码加随机数加加密算法生成的密码
    从1970年1月1日起到密码最近一次被更改的时间 echo date +%s/60/60/24 |bc
    要求用户几天更改密码：密码再过几天可以被变更（0表示随时可被变更）
    密码最长使用期：密码再过几天必须被变更（99999表示永不过期）
    密码警告期：密码过期前几天系统提醒用户（默认为一周）
    密码无效期：密码过期几天后帐号会被锁定
    用户到期日期：从1970年1月1日算起，多少天后帐号失效

    更改加密算法：

    authconfig –passalgo=sha256 –update

    /etc/gshadow：组密码及其相关属性

    组名：密码：组管理员列表：组成员

    usermod

    usermod 
-u UID: 新UID
-g GID: 新主组
-G GROUP1[,GROUP2,...[,GROUPN]]]：新附加组，原来的附加组将会被覆盖；若保留原有，则要同时使用-a选项
-s SHELL：新的默认SHELL
-c 'COMMENT'：新的注释信息
-d HOME: 新家目录不会自动创建；若要创建新家目录并移动原家数据，同时使用-m选项
-l login_name: 新的名字
-L: lock指定用户,在/etc/shadow 密码栏的增加 !
-U: unlock指定用户,将 /etc/shadow 密码栏的 ! 拿掉
-e YYYY-MM-DD: 指明用户账号过期日期
-f INACTIVE: 设定非活动期限

附加组
usermod -G root tom #添加一个root组，并清除以前的组
usermod -aG root tom #给tom用户添加一个root组
usermod -G “” tom 删除附加组  #usermod -G tom tom

将用户添加到组，有3种方法

    文件权限

    chmod 修改文件权限

    • 文件：
      r 可使用文件查看类工具获取其内容
      w 可修改其内容
      x 可以把此文件提请内核启动为一个进程
    • 目录：
      r 可以使用ls查看此目录中文件列表
      w 可在此目录中创建文件，也可删除此目录中的文件
      x 可以使用ls -l查看此目录中文件元数据（须配合r），可以cd进入此目录
      X 只给目录x权限，不给文件x权限
      read write excute
    模式法
chmod who opt per file
who:u,g,o,a=all
opt:+,-,=
per:r,w,x
file:

chmod 
chmod 777 down.sh
chmod u+x down.sh
-R: 递归

    注意：
    目录读权限，查看文件列表，当文件夹没有读权限，ls无法列出文件列表
    目录执行权限，无法进入目录，可以查看文件名，无法查看属性
    删除文件和文件的本身权限没有关系，和父目录权限有关wx（还需要年上级目录权限）
    cp f1 dir/f2
    命令的执行权限 文件的读权限（父目录要有执行权限）拷贝目录的文件夹写和执行权限 注意：文件名若已存在，无法执行成功，无法覆盖文件
    chmod -R +X 大X代表给目录加执行权限，使用场景不是很多
    man 5 chmod

3、当用户redis 对/testdir 目录无写权限时，该目录下的只读文件file1是否可修改和删除？ 
不可以删除，不可修改文件，注意（修改文件要看文件的权限）

4、用户zabbix对/testdir 目录有写和执行权限时，该目录下的只读文件file1是否可修改和删除？
不可修改，可以删除

5、复制/etc/fstab文件到/var/tmp下，设置文件所有者为tomcat读写权限，所属组为apps组有读写权限，其他人无权限？ 
cp /etc/fstab /var/tmp && chown tomcat.apps /var/tmp/fstab && chmod 660 /var/tmp/fstab

6、误删除了用户git的家目录，请重建并恢复该用户家目录及相应的权限属性 
cp -a /etc/skel /home/git && chown -R git.git /home/git && chmod 700 /home/git

    特殊权限

    suid—-以所属者的权限

    chmod u+s file
    chmod u-s file
    SUID: user,占据属主的执行权限位
    s：属主拥有x权限
    S：属主没有x权限
    注意：
    1、SUID SGID只对二进制可执行程序有效
    2、SUID对目录上无意义
    3、进程访问文件权限>>user>>group>>other
    4、任何一个是否可执行程序，取决于执行者上否拥有执行权限
    5、简单来说就加SUID SGID，执行程序时，会以所属者或所属组的权限执行，换个身份，仅限此执行程序，执行完毕又回来普通用户的身份
    6、额外补充（如果U盘集成VIM程序有特殊权限，也可以编辑/etc/shadow文件）
    cp -a /usr/bin/vim /tmp && chmod u+s /tmp/vim && su - user
    ll /etc/shadow && vim /etc/shadow && ./tmp/vim /etc/shadow

    sgid

    chmod g+s file
    chmod g-s file
    SGID: group,占据属组的执行权限位
    s： group拥有x权限
    S：group没有x权限
    默认建立文件所属组，自动继承
    1、支持在二进制的可执行程序上，执行程序时，会以所属组的权限执行
    2、还有一个作用在目录上，在此目录中新建的文件所属组，会自动继承目录的所属组

    sticky 对目录设置

    ll -d /tmp
    chmod o+t dir/
    Sticky: other,占据other的执行权限位
    t：other拥有x权限
    T：other没有x权限
    目录加了sticky权限 ，在这个目录下用户只能删除自己建立的文件，
    1、如果文件权限较大，可以修改并清空内容
    2、不同用户建立的文件其他用户不可删除

    设定文件特定属性

    chattr +i 不能删除，改名，更改
    chattr +a 只能追加内容
    lsattr 显示特定属性

    acl 访问控制列表

    当加硬盘时使用
    alias scandisk=‘echo - - - > /sys/class/scsi_host/host0/scan’ 命令重新扫描下磁盘（增加型）

    1、vfat磁盘格式在linux中有些功能没有，比如修改文件权限，文件的所属者，所属组
    2、ACL优先顺序：所有者，ACL用户，ACL组，ACL其他，（注意MASK第5条、第六条）ACL没有才看其他人#ACL设置不看组
    3、tar打包文件用丢失ACL权限，可以将ACL权限写到到文件，解压后添加
    4、这种情况基于属于灾难
    chmod -R 444 / 破坏力等于 rm -rf / 破坏力等于 chown -R user1
    5、不能超过mask权限 getfacl file 查看，如果后加的权限会覆盖，注意限高
    6、如果两种权限都有，注意优先级、顺序
    7、如果一个用户有多少组，权限累加

    清除文件acl权限信息，恢复默认
setfacl -b file1
setfacl -m u:user:rwx file 设置ACL
setfacl -x u:user: file 删除权限
getfacl file1 查看文件ACL权限
ls file1               #+号代表拥有ACL权限
-rwxrwx---+ 1 root root 1 1 1 1 file1 
如果一个用户拥有多个组，那拥有多个组的累加权限
将ACL权限写在文件中，使用命令调用
echo u:test:rw > acl.txt
setfacl -M alc.txt file1

getfacl f1 |setfacl --set-file=- f2 复制f1文件的acl权限,设置到f2上

mask是所有权限
sefacl -m make::rw f1

    ACL还可以这样用

    chmod -x /usr/bin/chmod
执行chmod时没有执行权限
setfacl u::x /usr/bin/chmod
chmod 755 /usr/bin/chmod && setfacl -b /usr/bin/chmod

    练习

    1、在/testdir/dir里创建的新文件自动属于webs组，组apps的成员如：tomcat能对这些新文件有读写权限，组dbs的成员如：mysql只能对新文件有读权限，其它用户（不属于webs,apps,dbs）不能访问这个文件夹
groupadd webs && mkdir -p /testdir/dir && chgrp webs /testdirr/dir && chown g+s /testdir/dir
usermod -aG apps tomcat  #组名在前 用户名在后 && setfacl -m g:apps:rw /testdir/test
groupadd dbs && useradd -aG dbs mysql
setfacl g:dbs:r /testdir/dir && setfacl g:webs:r /testdir/dir && chmod o= /testdir/dir

2、备份/testdir/dir里所有文件的ACL权限到/root/acl.txt中，清除/testdir/dir中所有ACL权限，最后还原ACL权限
getfacl -R /test/dir/* > /root/acl.txt
setfac -b /test/dir/*
setfacl -R -M //root/acl.txt /test/dir/*
