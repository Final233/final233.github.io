    定制linux

    内核名中间的数字是次数，编译了多少次才稳定了内核状态

    https://www.kernel.org/

    编译内核

    查看内核参数，哪些功能开启

    less /boot/config-3.10.0-957.el7.x86_64

    1、下载内核包

    wget https://cdn.kernel.org/pub/linux/kernel/v5.x/linux-5.7.4.tar.xz

    2、解压内核包，确保磁盘空间足够，CPU处理越快，编译时间越短

    tar xf linux-5.7.4.tar.xz

    3、安装依赖包

    yum install gcc gcc-c++ glibc glibc-devel pcre pcre-devel openssl openssl-devel systemed-devel zlib-devel -y

    4、复制memuconfig文件到刚解压的内核目录下(在原有内核基础上进行添加或修改)

    cp /boot/config-3.10.0-957.el7.x86_64  linux-5.7.4/.config

    5、编译 make menuconfig

    根据实现情况来安装依赖包
发现少安装一个包 yum install -y ncurses-devel
yum search bison
yum install -y biosn-devel bison flex
yum install -y  elfutils-libelf-devel

    6、编译 make menuconfig
    General setup —-Local verison - append to kernel release
    -1.0-final233
    exit
    File system —- DOS/FAT/NF Filesystems —- NTFS file system suport —-
    NTFS write support
    M 模块方式 * 打入内核 为空的话 不启用此功能
    exit exit
    保存退出








    退出 exit
    7、编译 make -j 8 && echo -e ‘\a’
    8、make modules_install 将编译好的模块文件，复制到/lib/modules/目录下
    9、make install 安装
    10、修改grub引导,根据引导填写正确的引导文件位置

    grub2-mkconfig -o /boot/grub2/grub.cfg

    11、重新启动进入系统

    uname -r 查看内核版本
5.7.4-0.1-final233

    12、卸载刚才安装的内核

    rm -f /boot/initramfs对面的版本删除 vmlinuz-xx
rm -rf /lib/modules/5.7.4-0.1-final233/
rm -rf linux-5.7.4/ 编译的目录

make distclean 清理

    5、格式化分区

    mkfs.ext4 /dev/sdb1

    6、挂载分区

    [root@node ~]# mkdir /mnt/boot

    5、安装grub2 (注意 grub与grub2有定的区别，机制和原理不太一样)

    [root@localhost ~]# grub2-install --root-directory=/mnt /dev/sdb

    6、复制启动相关文件

    cp vmlinuz-5.7.4-0.1-final233 initramfs-5.7.4-0.1-final233.img /mnt/boot/

    7、写grub文件

    default=0

initrd /initramfs-5.7.4-0.1-final233.img

    6、创建相关目录

    mkdir /mnt/root
mount /dev/sda2 /mnt/root
for DIR in `ls /`;do mkdir /mnt/root/$DIR;done

    7、添加相关的命令及运行库（注意 cp复制软链接原文件cp -L）

    mkdir /mnt/sysroot
bash copycmd.sh command
ls /bin/|while read cmd ;do sh copy_cmd.sh ;done
ls /sbin/|while read cmd ;do sh copy_cmd.sh ;done
cp -a /mnt/sysroot/lib64/ /mnt/root/
cp -a /mnt/sysroot/usr/ /mnt/root/

    Busybox介绍

    • Busybox 最初是由 Bruce Perens 在 1996 年为 Debian GNU/Linux 安装盘编
      写的。其目标是在一张软盘(存储空间只有1.4-1.7MB)上创建一个可引导的
      GNU/Linux 系统，可以用作安装盘和急救盘

    • Busybox 是一个开源项目，遵循GPL v2协议。Busybox将众多的UNIX命令集
      合进一个很小的可执行程序中，可以用来替代GNU fileutils、shellutils等工具
      集。Busybox中各种命令与相应的GNU工具相比，所能提供的选项比较少，但
      是也足够一般的应用了。Busybox主要用于嵌入式系统

    • Busybox 是一个集成了三百多个最常用Linux命令和工具的软件。BusyBox 包
      含了一些简单的工具，例如ls、cat和echo等等，还包含了一些更大、更复杂的
      工具，例grep、find、mount以及telnet。有些人将 BusyBox 称为 Linux 工具
      里的瑞士军刀。简单的说BusyBox就好像是个大工具箱，它集成压缩了 Linux
      的许多工具和命令，也包含了 Android 系统的自带的shell

    • 定制小型的Linux操作系统：linux内核+busybox

    • busybox 的编译过程与Linux内核的编译类似

    • busybox的使用有三种方式：

      busybox后直接跟命令，如 busybox ls

      直接将busybox重命名，如 cp busybox tar

      创建符号链接，如 ln -s busybox rm

    • busybox的安装
      以上方法中，第三种方法最方便，但为busybox中每个命令都创建一个软链接，
      相当费事，busybox提供自动方法：busybox编译成功后，执行make install,则会
      产生一个_install目录，其中包含了busybox及每个命令的软链接

      https://busybox.net/

      yum install -y  elfutils-libelf-devel
wget https://busybox.net/downloads/busybox-1.31.1.tar.bz2
tar xf busybox-1.31.1.tar.bz2 
cd busybox-1.31.1
make menuconfig  等于 ./configure --xxxxxxxx

      setting—- build options —- build static binary (no shared libs) #所有的库不是共享的，是集成在二进制 程序里面
      yum install glibc-static

      不共享库，把库集成在一个二进制程序中，独立的所有的库不是共享的，是集成在二进制程序里面

      make -j4
make install

      会在当前目录下生成一个目录并安装在此目录下_install

      cd busybox-1.31.1/_install
tree .
cp -av . /mnt/root/enuconfig 按下面选择，把busybox编译也静态二进制、不用共享库
Busybox Settings -->Build Options -->[*] Build BusyBox as a static
binary (no shared libs)

      make && make install 如果出错，执行make clean后，重新执行上面命令

      mkdir /mnt/sysroot/
cp -a _install/* /mnt/sysroot/

    总结

    有几点注意事项

    1、copycmd.sh脚本注意lib库的文件和文件，复制原文件，不是软链接文件

    2、注意网卡驱动的位置，系统启动，加载网卡模块

    3、busybox编译后的文件复制到相对应的目录下

    4、bash复制过去后，切换根（chroot /mnt/sysroot）目录调试下

    5、grub/grub2有一定的区别，grub2注意下引导配置文件，grub2引导机制有点不同

    6、mbr/efi引导注意分区和引导文件的位置

    7、常用的命令使用copycmd.sh脚本生成

    8、busybox+定制LINUX（编译后的内核+常用命令+驱动模块+等）

    9、定义了内核功能加载项

    linux-xxxxxxxx/.config
    y表示放在内核中
    m表示是模块，需要时加载，不需要不加载

    10、rpm -l kernel /boot目录下编译后安装的文件 /lib模块

    11、加载模块
    modprobe 会在只指定的目录下加载，会解决模块的依赖关系，自定义内核或linux要注意
    insmod 需要指定加载模块的完整路径才可以,并不会自动去搜索

    12、编译内核注意gcc，有些新版本的内核，需要新版的gcc gcc-c++
